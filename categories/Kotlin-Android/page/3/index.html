实习笔记-25

创建型

单例

优点

  • 节省资源
    • 内存
    • 重对象中包含的io，文件指针
  • 调用方便

缺点

  • 适用于昂贵对象，对于轻量对象，为了维护单例造成的同步等开销比创建一个对象更高。得不偿失
  • 不方便mock，（可以把单例对象作为函数参数则可以mock），相较于静态方法更方便（静态方法需要代理的方式mock）
  • 测试时常常并行测试，使用单例会降低效率
    函数式编程，拷贝
    云控，打点 Alex
    静态工具方法，少用，不要有状态，确保永远不会变
